# Bluetooth Low Energy (BLE)

## 概要

昔からある Bluetooth を Bluetooth Classic と (後付けで) 呼ぶ。
新しい低消費電力の規格を Bluetooth Low Energy (BLE) と呼ぶ。

2.4 GHz 帯 の無線。

* 1999 Bluetooth Classic
* 2009 Bluetooth Low Energy (Bluetooth 4.0)

なお名前は似ているが、規格の中身は概念から全然違う模様。

Bluetooth Classic といえば無線ヘッドフォンらしい。
BLE は Classic と比べると消費電力が 1/30 とかそういうレベルらしい。
当然モバイルスマート系デバイスと相性がよい。

## BLE 用語

あまり他の情報通信技術では使わない用語が多い気がする。
とっつきが悪いが我慢すること。

### デバイスとアドバタイズ

Bluetooth デバイスを識別する MAC address みたいな見た目のアドレスを
**BD Address** / **BD_ADDR** (Bluetooth Device Address) と呼ぶ。

親側を **セントラル** 子側を **ペリフェラル** と呼ぶ。

ペリフェラルがセントラルからの接続を待つため、自身の存在を示す信号を発信することを
**アドバタイズ** と呼ぶ。
advertise = 広告する、宣伝する。

アドバタイズを受信することを **スキャン** と呼ぶ。
近くにある Bluetooth 機器一覧を表示する、みたいなアレ。使ったことがあるはず。

セントラルがアドバタイズ信号を受信したとき、
受信した電波強度情報を手に入れることができる。
これを **RSSI (Received Signal Strength Indicator)** と呼ぶ。

### GATT

セントラルは見つけたペリフェラルの中から一対一通信をしたい相手を選び、
接続要求を送信することができる。
この一対一の通信を **GATT** と呼ぶ。
GATT = Generic ATTribute profile

GATT では **サービス** と **キャラクタリスティック** という概念を用いる。

サービスは複数のキャラクタリスティックを一つの機能単位にまとめた
ラベルのようなものである。

それぞれのキャラクタリスティックは **Read** **Write** **Notify** のような属性を持ち、
セントラルは許された操作を行いデバイスの機能を使うことができる。

サービスとキャラクタリスティックにはユニークな **UUID** がついており、
これで対象を指定する。

### ペアリング

**ペアリング** は "相手を覚えておくこと" 全般を指す、
意味のぼやけた用語になってしまっている。
正式な技術用語としては **ボンディング** の方が適切らしい。
概ね以下のうちのどれかまたは複数をぼんやり指すことが多い。

* ペリフェラルに接続すること
* BD_ADDR を覚えて次回に自動接続したりすること
* お互いにしか分からない暗号鍵を共有して、暗号化したセキュアな通信路を確立すること

以下、セキュア通信のためのペアリングについて説明する。

* LE Legacy Pairing
  * Bluetooth 4.0、つまり最初に導入された方法。
  * 脆弱性が見つかったので使ってはいけない。
* LE Secure Connections (LESC)
  * 脆弱性を受けて Bluetooth 4.2 で制定されたまともな方式。
  * Legacy と違って公開鍵暗号、楕円曲線暗号、DH 鍵交換など、
    マトモな感じになっているっぽい。

BLE 接続が確立した後、ペアリング要求と応答が行われる。
この時、必要に応じて認証が行われる。
特に何の確認もなく OK とするデバイスもある。

認証が成功したら、暗号鍵の鍵交換が行われる。
このような暗号化情報を不揮発性領域に保存し、次回にも同じ情報を使ったりする。
これを **ペアリング情報** と呼ぶ。

## コマンドラインツール

昔はいろいろあったのかもしれないが、Linux では `BlueZ` というドライバ・
プロトコルスタックがおすすめらしい。
なお、それはそれでツールのパッケージ移動や非推奨化があったりでやや謎。

```sh
# だいたいこれで OK
apt install bluez
# 足りなかったら
# bluez-tools bluez-utils
# みたいな名前のやつを入れてみる
```

`hciconfig` みたいな名前のやつは非推奨らしい。

```sh
# stdin からの対話モードに入る
bluetoothctl
# stdin を使っているのでリダイレクトで自動化できる
bluetoothctl < input.txt
# コマンドを1つだけ実行して終了する
bluetoothctl show
```
